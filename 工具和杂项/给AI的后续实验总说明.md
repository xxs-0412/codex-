# 给 AI 的后续实验总说明

## 1. 这份文档的用途

这份文档的目标是让任何一个新接手的 AI 或协作者，在**不依赖上下文对话**的情况下，也能快速理解这个项目当前已经做到哪里、后面准备怎么继续做，以及每一步实验之间的继承关系。

这份文档不是固定死所有实验细节，而是固定：

- 实验主线
- 数据管理方式
- 结果产物结构
- 各类实验之间的先后顺序与依赖关系

以后具体超参数、约束形式、特征组合还可以继续讨论和调整，但大方向应保持一致。

---

## 2. 项目目标

本项目围绕“涂层型关节轴承寿命预测”展开，核心思路是：

1. 由模型预测接触应力或应力相关量
2. 结合递推磨损逻辑计算磨损演化
3. 当磨损达到阈值时得到寿命预测结果

后续实验的主要目标不是随意堆新模型，而是沿着以下四条实验线逐步推进：

1. **基准测试**
2. **Transformer 升级测试**
3. **Shape Loss / 损失函数约束测试**
4. **特征构造升级测试**

这四条线的先后顺序是有依赖关系的，不能打乱。

## 4. 后续实验的总主线

以后无论是换数据，还是从留一验证改成固定测试集验证，整体顺序都应遵循：

1. **先确定这轮实验使用的数据**
2. **先做基准测试**
3. **再做 Transformer 升级测试**
4. **在“本轮最佳 Transformer”基础上做 Shape Loss 测试**
5. **在“本轮最佳 Transformer”基础上做特征构造升级测试**

换句话说：

- `shape_loss` 不是从旧 baseline 直接起步
- `特征构造` 也不是从旧 baseline 直接起步
- 它们都应当继承“同一轮数据、同一套测试划分下”的**最佳 Transformer 基线**

这是后续实验最重要的继承关系。

## 5. 数据管理规则

### 5.1 累计库与快照分离

当前根目录里有两类数据目录：

- 累计库  
  - `提取结果`
  - `提取结果_处理后`

- 冻结快照  
  - `数据快照/旧版留一_4.13`
  - `数据快照/新17run_4.18`
  - `数据快照/固定测试集试验_v1`

规则如下：

1. `提取结果` / `提取结果_处理后` 用来**持续追加新数据**
2. 每次正式开新一轮实验前，要把本轮要用的确切数据复制到一个新的 `数据快照`
3. 正式实验脚本应优先读取快照，不直接读取累计库

### 5.2 为什么要这样做

因为以后你还会继续加数据。如果直接让实验脚本始终读累计库，那么：

- 老实验可能被新数据污染
- 结果不容易复现
- 后面很难说清楚某一组结果到底用了哪一批数据

所以：

- **累计库**负责“长期积累”
- **快照**负责“本轮实验冻结”

### 5.3 以后新增数据时的默认流程

当新数据到来时，默认流程应该是：

1. 把新数据追加到 `提取结果`
2. 完成处理后，追加到 `提取结果_处理后`
3. 若准备正式做新一轮实验，则新建一个快照目录，例如：
   - `数据快照/固定测试集试验_v2`
   - 或 `数据快照/新24run_xxxx`
4. 从累计库复制这轮要用的数据到新快照
5. 后续所有脚本都指向这份新快照

## 7. 后续四类实验的详细说明

## 7.1 第一类：基准测试

### 目标

先建立“本轮数据 + 本轮测试集划分”下的共同对照组。

### 为什么必须先做

因为后续所有结果都需要知道：

- 在同一批数据下
- 在同一套测试集划分下
- 各基础模型的表现到底如何

否则后面 Transformer 提升了多少、Shape Loss 有没有价值、特征构造有没有价值，都没有统一参照。

### 默认做法

建议继续保留五模型对比：

- FNN
- GRU
- LSTM
- 1D-CNN
- Transformer baseline

### 输出

至少应输出：

- 详细结果 CSV
- 汇总 CSV
- 对比图
- 测试说明
- 结果分析

### 结果作用

这一步的 **Transformer baseline** 是后续升级测试的起点。

---

## 7.2 第二类：Transformer 升级测试

### 目标

在“本轮基准测试里的 Transformer baseline”之上，寻找更优的 Transformer 配置。

### 当前已验证过的有效方向

根据已经完成的 `transformer升级测试v2`，目前验证过的总体方向包括：

- 调整 `seq_len`
- 调整 `d_model`
- 调整前馈层维度 `ff`
- 调整层数
- 重新评估训练策略与正则化分支

### 当前已知经验

在当前 17-run 数据上，已经观察到：

- 单纯强化正则化和复杂训练策略不一定有利
- 序列长度是首先见效的变量
- 适度增加模型容量通常比增加过深层数更有效

### 当前最佳正式结果

当前正式最佳结果来自：

- `结果/transformer升级测试v2`

最终正式版参数方向为：

- `seq_len=15`
- `d_model=64`
- `nhead=4`
- `layers=2`
- `ff=128`

### 后续阶段的继承规则

以后如果换数据或换测试集划分，**不能直接默认沿用当前参数为最终答案**，而是要：

1. 先重新跑本轮 Transformer 升级测试
2. 在本轮数据/划分下重新选出最佳 Transformer
3. 再把这个“本轮最佳 Transformer”作为后续增补模块的 baseline

也就是说：

- 当前 `transformer升级测试v2` 是一个很强的参考起点
- 但在新数据/新划分下，仍应重新验证，而不是无条件照搬

---

## 7.3 第三类：Shape Loss / 损失函数约束测试

### 目标

在本轮最佳 Transformer 基础上，引入额外的物理/时序软约束，观察是否能进一步改善：

- 压力拟合
- 磨损曲线质量
- 寿命误差

### 当前已经明确的方向

根据现有 `损失函数Loss试验.md`，这条线的核心方向是：

- 在原有主损失之外，加入同一 run 内的组内时序约束
- 约束作用在**物理 stress 空间**
- 只在低磨损区间（`wear_depth <= 0.005 mm`）内生效

当前明确的两类约束是：

1. `run_stress_mono_lambda`
   - 约束同一 run 内应力应单调递减
   - 若 `s_{t+1} > s_t`，则产生惩罚

2. `run_stress_slow_lambda`
   - 约束应力下降幅度应递减
   - 即越往后下降越慢

### 当前建议的实验组织方式

不要把 Shape Loss 混进大规模参数搜索主流程里，而是做一个**独立对照实验**。

标准对照组可以是：

- `baseline`
- `shape_loss_v1`
- `shape_loss_v2`

其中：

- `baseline`：不开 shape loss
- `shape_loss_v1`：只开 mono
- `shape_loss_v2`：开 mono + slow

### 当前推荐的继承方式

如果进入新的固定测试集阶段，Shape Loss 不应直接继承旧的 `seq=10` 或旧版人工配置，而应继承：

- **本轮固定测试集下重新得到的最佳 Transformer baseline**

### 允许继续讨论的部分

这条线未来可以继续讨论和调整的，不是主线顺序，而是这些细节：

- mono / slow 的权重大小
- 约束生效区间
- 是否只约束前期
- 是否针对长寿命工况做特殊处理
- 是否增加其他辅助约束

但实验组织方式仍建议保持：

**先有最佳 Transformer baseline，再做独立的 shape loss 对照。**

---

## 7.4 第四类：特征构造升级测试

### 目标

通过改造输入特征，让模型更容易学习磨损演化过程中的物理模式和时间结构。

### 当前已经明确的方向

根据现有 `特征构造升级.md`，这条线的主要思路是从原始 5 维输入出发，逐步增加更有物理意义的派生特征。

当前基础 5 维特征为：

- `F`
- `D`
- `Cr`
- `actual_cycle`
- `wear_depth`

当前已经明确的升级方向包括：

#### 静态派生特征

- `F / D^2`
- `Cr / D`

#### 动态变换特征

- `log1p(actual_cycle)` 代替原始 cycle

#### 差分特征

- `delta_cycle`
- `delta_wear`

### 当前建议的实验路线

已有较清晰的分轮次思路：

- `baseline`：原始 5 维
- `R1`：加入静态派生特征
- `R2`：在静态派生基础上，把 `cycle` 改成 `log1p(cycle)`
- `R3`：继续加入 `delta_cycle`、`delta_wear`
- `R4`：从前面几轮里选择最优组合作为正式方案

### 当前推荐的继承方式

和 Shape Loss 一样，这条线也应继承：

- **本轮固定测试集下重新得到的最佳 Transformer baseline**

而不是直接拿旧版 `leave-one-run` 的最佳配置不加验证地硬套。

### 允许继续讨论的部分

这条线未来可继续讨论的内容包括：

- 最终保留哪些特征
- 是否保留原始 `cycle`
- 是否只用 `log1p(cycle)`
- 是否增加更多静态组合量
- 差分特征是否只给 Transformer 使用
- 是否让不同模型共用同一套特征

但实验顺序仍应保持：

**先有本轮最佳 Transformer baseline，再做特征构造升级对照。**

## 8. 以后换数据或换测试集时的标准流程

以后每一轮新实验，建议都按下面的固定流程执行：

### 第一步：更新累计库

把新数据加入：

- `提取结果`
- `提取结果_处理后`

### 第二步：冻结本轮数据快照

把本轮正式要用的数据复制到一个新快照目录，例如：

- `数据快照/固定测试集试验_v2`

### 第三步：确定测试集划分

明确四组固定测试集清单，例如：

- `split_A`
- `split_B`
- `split_C`
- `split_D`

### 第四步：先跑基准测试

在本轮固定测试集上重新得到五模型基准结果。

### 第五步：再跑 Transformer 升级测试

在本轮数据和本轮划分下重新选出最佳 Transformer。

### 第六步：再跑两个增补模块

分别进行：

- Shape Loss 对照试验
- 特征构造升级试验

两者都继承本轮最佳 Transformer baseline。

### 第七步：统一写说明

每一轮都应把以下内容写清楚：

- 使用了哪份数据快照
- 使用了哪套 split
- 本轮最佳 baseline 是什么
- Shape Loss / 特征构造继承的是哪一版 baseline
