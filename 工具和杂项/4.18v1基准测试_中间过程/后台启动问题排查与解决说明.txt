后台启动问题排查与解决说明
============================================================

一、问题现象
1. `结果/4.18v1基准测试/run_benchmark.py` 前台直接运行可以正常启动，并且能识别 CUDA 和 `NVIDIA GeForce RTX 5070 Ti`。
2. 但尝试通过后台方式启动时，经常出现“进程秒退、无日志、无结果文件”的情况。
3. 同类问题也会影响 Phase 1 的 Transformer v2 流水线后台执行。

二、排查到的具体原因
1. 原工作区路径改名之前包含中文，后台命令在部分启动方式下对中文路径和相对路径不稳定。
2. 当前 PowerShell 会话环境里同时存在 `Path` 和 `PATH` 两个环境变量，直接调用 `Start-Process` 时会触发：
   `已添加项。字典中的关键字:“Path”所添加的关键字:“PATH”`
3. 早期补的 `.cmd` 启动器使用了 LF 换行，`cmd.exe` 在 Windows 下会错误拆行，导致批处理本身执行失败。
4. 即使脚本、路径和环境变量问题都修过，当前 Codex shell 工具环境下的“后台脱离进程”仍然不稳定，shell 返回后子进程有较大概率被回收，所以表现为：
   - 前台能跑
   - 后台秒退
   - 日志文件还没来得及稳定落盘

三、已经做过的修复
1. 新建了 ASCII 别名入口，避免后台启动链路直接依赖中文脚本路径：
   - `result_benchmark_418_v1`
   - `result_transformer_upgrade_v2`
   - `tool_benchmark_418_v1_intermediate`
   - `tool_transformer_upgrade_v2_intermediate`
2. 新写了 PowerShell 启动器，替代不稳定的 `.cmd` 方案：
   - `工具和杂项/4.18v1基准测试_中间过程/start_benchmark.ps1`
   - `工具和杂项/transformer升级测试v2_中间过程/start_transformer_pipeline.ps1`
3. 启动前会移除会话中的 `Env:PATH`，避免 `Start-Process` 因 `Path/PATH` 冲突直接报错。
4. 启动器内部改成“从脚本路径向上探测工作区根目录”的方式定位项目根目录，而不是依赖容易漂移的相对工作目录。

四、最终采用的解决方案
1. 不再依赖当前工具环境里的“后台常驻子进程”来跑正式训练。
2. Phase 1 正式训练改为同步长跑执行。
3. 为避免同步长跑中断后前功尽弃，已经给以下三个正式入口加入“断点续跑/增量落盘”能力：
   - `结果/4.18v1基准测试/run_benchmark.py`
   - `工具和杂项/transformer升级测试v2_中间过程/run_all_rounds.py`
   - `结果/transformer升级测试v2/run_benchmark.py`
4. 新逻辑下，已经完成的折次或候选轮次会先写入对应 CSV；如果训练因超时或中断停止，下次重新执行时会自动跳过已完成部分。

五、当前结论
1. 问题不是 CUDA、不是模型脚本本身，也不是新数据有问题。
2. 真正的问题是“当前工具环境下后台脱离执行不可靠”，叠加了：
   - 中文路径链路
   - `Path/PATH` 环境变量冲突
   - `.cmd` 换行格式不适配
3. 对本轮实验最稳妥的方案，就是：
   - 保留 ASCII 启动别名和 PowerShell 启动器
   - 正式实验使用同步执行
   - 依靠断点续跑保障长任务安全
