## 5. 关键代码片段或关键逻辑变化说明

### 5.1 仿真周期换算为实际周期

来源：`process_real_wear_data.py`

```python
actual_cycle_float = sim_cycle * k_sim / real_k
actual_cycle = strict_round_cycles(actual_cycle_float)
```

说明：

- 严格按用户要求的比例关系换算；
- 再做整数化与单调递增修正。

### 5.2 缺失压力时反推压力

来源：`process_real_wear_data.py`

```python
delta_s = delta_sim * math.pi * float(diameter_mm) / 6.0
stress = delta_wear / np.maximum(float(k_sim) * delta_s, EPS)
```

说明：

- `Run3.xlsx` 没有 `CPRESS`，所以这里用 Archard 公式反推等效压力。

### 5.3 阈值寿命估计

来源：`process_real_wear_data.py`

优先级：

1. 若原始曲线已到阈值：插值求寿命
2. 若未到阈值：用

```python
y = a*x + b*(1 - exp(-c*x))
```

拟合外推
3. 若拟合失败：退回尾段线性外推

### 5.4 训练中引入物理单调性软约束

来源：`train_real_wear_models.py`

关键思路：

- 不新增训练样本文件；
- 对同一条真实样本，临时构造几种小扰动版本：
  - `h + Δh`
  - `F * (1 + Δ)`
  - `Cr + ΔCr`
- 然后把这些趋势写进损失函数：

```python
wear_penalty = torch.mean(torch.relu(pred_wear_plus - pred) ** 2)
load_penalty = torch.mean(torch.relu(pred - pred_load_plus) ** 2)
clearance_penalty = torch.mean(torch.relu(pred - pred_clearance_plus) ** 2)

loss = (
    data_loss
    + MONO_LAMBDA_WEAR * wear_penalty
    + MONO_LAMBDA_LOAD * load_penalty
    + MONO_LAMBDA_CLEARANCE * clearance_penalty
)
```

说明：

- 这是“用损失函数教物理规律”，不是“往数据集里加虚拟样本”。

### 5.5 递推模型与静态模型的根本区别

递推模型：

- 先预测单步压力
- 再用物理公式更新磨损
- 再递推寿命

静态模型：

- 直接输入 `[F, D, Cr]`
- 直接输出一个终值寿命

### 5.6 多网络结构 benchmark

- LSTM
